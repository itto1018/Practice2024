# 1-1.よいWebデザインとは
### デザインの目的
相手に伝えること(伝える手段であり、装飾することではない)
# 1-2.サイトの種類
### ポイント：目的に合わせて構成する
種類|説明
-|-
**コーポレートサイト**|企業についての情報発信
**プロモーションサイト**<br>(特定サイト)|特定の商品やサービス、イベントの告知に使う<br>ターゲットを絞って発信,期間を絞って発信
**ポートフォリオ**|自身の作品や制作実績を掲載
**ショッピングサイト**<br>(ECサイト,オンラインストアサイト)|商品の販売
**メディアサイト**|ニュースや読み物記事で構成
**SNS**|インスタグラムなど、記事として掲載するほどでもない短文や画像・動画をメインとした情報発信

# 1-3.ユーザビリティとは
### 使いやすさを考える
ユーザビリティとは、使いやすさのこと
### 見やすいデザインにする
- **色使いに注意する**<br>1. 背景色と文字色のコントラスト(明るさに大きく差をつける)<br>2. 目に優しい色を使う<br>
コントラストがついてるか確認するには、<span style='color: lightgreen; font-weight: bold;'>グレースケール</span>にする
- **目立たせたいものを明確にする**<br>目的に応じてテキストやボタンのサイズに差をつける
- **レイアウトを統一する**<br>ロゴやナビゲーションメニュー、フッターなどの共通部分はデザインを統一する
### 読みやすい文章にする
- 結論を先に書く
- 専門用語を避ける<br>どうしても使いたい場合は、注釈を入れる
- 簡潔にまとめる
### 使いやすい操作性にする
- 予想ができるようにする<br>押すと何が起こるかわかるボタンにする(ロゴを押すと、HPに遷移するなど)
- 動作を早くする
- 一目見てわかるようにする
# 1-4 Webサイトの仕組み
### WebサーバとWebクライアントの違い
- Webサーバ：Web上で情報を公開しており、ファイルの保存機能等はあるが、操作するための機能がついていない
- Webクライアント：Webサーバから情報を受け取る側、ユーザが利用するコンピュータのこと
### URLの構成
プロトコル(http) + ドメイン名(~.com) + ディレクトリ名(~/~) + ファイル名 + ファイル拡張子
# 1-5 デバイスの種類
## 端末
### モバイルデバイス,ポータブルデバイス,スマートデバイス
- iOSデバイス
- Androidデバイス
### ウェアラブルデバイス
### IoTデバイス
## 周辺機器
USBデバイス, ストレージデバイス, オーディオデバイスなど
# 1-6 ブラウザーの種類
開発前にあらかじめテストで使う標準ブラウザを決めておくといい
- シェア率：Chrome > Safari > Edge
# 1-7 制作の流れ
No.|工程|説明
-|-|-
1|企画立案|目的・必要なコンテンツ・ターゲットを考える
2|サイトマップ制作|Webサイトの構成(どのページがどこにリンクしているか)
3|ワイヤーフレーム制作|テキストや画像、仕切り線などを簡素なラインとボックスで作成<br>必要な項目や優先順位、配置などの確認
4|デザイン|デザインツールを使い、デザインカンプを作成
5|コーディング|HTML/CSSなど
6|Web上に公開|公開後にも正しい挙動をするか確認する,運用・保守など
## 1.企画を立てる
### 目標を立てる
- メインゴールを立てる
- サブゴール(メインゴールを達成するための目標)を複数立てる
### ターゲットの選定
性別、年齢層(10歳ずつ)、職業、趣味など、ターゲットを明確にする
## 2.サイトマップ制作
### ページをグループ分けする
- なるべく2階層以内に収める
- 優先度の高いページはナビゲーションバーなどから遷移できるようにする
- ホワイトボードなどのツールを使ってもいいかも<br>Cacoo、イラレ、Excel、Figmaなど
## 3.ワイヤーフレーム制作
デザインはしないこと！装飾は極力加えずに必要なコンテンツをどう配置するかを重点的に考える<br>また、レスポンシブも考慮してモバイル用のワイヤーフレームも作成しておく
- コンテンツの優先順位<br>LPで何を伝えたいかわかるように
- 視線の動きを考える<br>上から下、左から右に視線を動かすことを意識する
- ワイヤーフレーム作成ツール(サイトマップ制作で使うツールと同じで良い)
## 4.デザイン
ワイヤーフレームを元に、デザインカンプを作成する<br>
もしデザインに困ったら、ギャラリーサイトを見てみよう(SANKOU!,I/O 3000,Awwwardsなど)
## 5.コーディング
### 初めにファイルを用意する
HTML/CSS,imgなど
- svgファイルは高解像度ディスプレイに対応させたいときに使う(ファイル自体は比較的重たい)
## 6.Web上に公開
1. サーバを用意する
2. ドメインを取得する
# 1-8 制作をはじめる前に
- テキストエディタ(VSCode)
- Webブラウザ(Chrome)
- グラフィックツール(Figma)

# 3-1 CSSとは
CSSとは、文書の見た目を装飾するための言語
# 3-2 CSSを適用させる方法
## 外部スタイルシート(linkタグ,rel属性,CSSファイルを読み込んで適用させる)
~~~
<!doctype html>
<html lang='ja'>
    <head>
        <link rel="stylesheet" href="style.css">
    </head>
</html>
~~~
## 内部スタイルシート(styleタグ)
~~~
<!doctype html>
<html lang='ja'>
    <head>
        <style>
            h1 { color: red; }
        </style>
    </head>
</html>
~~~
## インラインで記述(styleタグ)
~~~
<body>
    <h1 style="color: red;">Hello, World!</h1>
</body>
~~~
# 3-3 CSSファイルを作ろう
冒頭に文字コードを記述する
~~~
@charset "UTF-8";
~~~
# 3-4 CSSの基本の書き方を身につけよう
## CSSの基本文法
セレクター(h1)・プロパティ(color)・値(red)
### 複数のセレクターに指定する
~~~
h1, p { color: red; }
~~~
